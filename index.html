Components

It always start with a kind

“ A component is an Enyo kind that can publish properties, expose events, and contain other components. ”

enyo.kind({
    name: "SithLord",
    kind: "Jedi",
    events: { onSpeak: “” }, //on ne catch pas l'event => bubble up
    scream: function(message) {
        doSpeak({
            message: message,
            tone: "whiney" });
    })
});

Components

Encapsulating them

enyo.kind({
    name: "SithLords", // y'a un 's'
    components: [
        { kind: SithLord, name: "DarthVader",
            onSpeak: “listenToToys” },
        { kind: SithLord, name: "Palpatine" },
        { kind: SithLord, name: "DarthSidious" },
        { kind: Droid, name: "Grievous"}
    ],
    listenToToys: function(inSender, inEvent) {
         alert(inEvent.mesage);
    }
});

Why Enyo ?

vs. Sencha

  • Truly cross-platform
    • Sencha has separate frameworks for desktop (ExtJS) and mobile (Sencha Touch)
  • Free and open-source
    • Sencha is available under GPL and free commercial licenses, but most customers pay for Sencha in some way
  • Extensible & community-centric
    • Sencha has significant mindshare, but little community participation and a minimal ecosystem.
QConSF2012 - Peter Helm and DaveFreeman -Enyo-A-Truly-Cross-Platform-JavaScript-App-Framwork

Why Enyo ?

vs. JQuery

  • jQuery: Progressively enhance web sites
    • Doesn’t help with application structure
    • Doesn’t facilitate native-like app UI
  • Enyo: Build native-quality HTML5 apps
    • Helps write maintainable, reusable code
    • Delivers native-like polish & performance
